# 📘 دليل الإعداد التفصيلي

<div dir="rtl">

دليل خطوة بخطوة لإعداد موقع المسابقة الثقافية وتشغيله بالكامل.

---

## الجزء 1: إعداد Supabase

### الخطوة 1: إنشاء حساب

1. اذهب إلى https://supabase.com
2. اضغط **Start your project**
3. سجّل الدخول باستخدام GitHub أو Google أو البريد الإلكتروني

### الخطوة 2: إنشاء مشروع جديد

1. اضغط **New Project**
2. املأ الحقول:
   - **Name**: `quiz-app` (أو أي اسم تختاره)
   - **Database Password**: كلمة مرور قوية (احفظها)
   - **Region**: اختر الأقرب (مثلاً Frankfurt للشرق الأوسط)
3. اضغط **Create new project**
4. انتظر دقيقتين حتى يتم إعداد المشروع

### الخطوة 3: تشغيل سكريبت قاعدة البيانات

1. من القائمة الجانبية، اضغط على أيقونة **SQL Editor** (شكل </>)
2. اضغط **New query**
3. افتح ملف `database/schema.sql` من المشروع بأي محرر نصوص
4. انسخ **كامل** محتواه
5. الصقه في نافذة SQL Editor
6. اضغط زر **Run** (أو Ctrl+Enter)
7. يجب أن تظهر رسالة "Success. No rows returned"

### الخطوة 4: نسخ بيانات الاتصال

1. اذهب إلى **Project Settings** (أيقونة الترس ⚙️ بالأسفل)
2. اختر **API** من القائمة الجانبية
3. ستجد قسمين مهمين:

#### Project URL
انسخ الرابط كاملاً مثل:
```
https://abcdefghijk.supabase.co
```

#### anon / public key
انسخ المفتاح الطويل (يبدأ عادة بـ `eyJ...`)

⚠️ **انتبه**: لا تشارك أبداً مفتاح `service_role` (الخدمة) - مفتاح `anon` فقط هو الآمن للمتصفح.

### الخطوة 4.5: إعداد Storage لرفع الصور 🖼️

لرفع صور الأسئلة مباشرة من لوحة الإدارة:

1. من القائمة الجانبية في Supabase، اضغط على **Storage** (أيقونة الصندوق 📦)
2. اضغط **New bucket**
3. املأ:
   - **Name**: `quiz-images` (بهذا الاسم بالضبط)
   - **Public bucket**: ✅ فعّل هذا الخيار (مهم جداً!)
   - File size limit: `5 MB` (اختياري)
   - Allowed MIME types: `image/*` (اختياري)
4. اضغط **Save**

#### إضافة سياسات Storage:

اذهب إلى **SQL Editor** ونفّذ:

```sql
-- السماح بالقراءة العامة من bucket quiz-images
CREATE POLICY "Public read quiz-images"
ON storage.objects FOR SELECT
USING (bucket_id = 'quiz-images');

-- السماح برفع الصور (للوحة الإدارة)
CREATE POLICY "Public upload quiz-images"
ON storage.objects FOR INSERT
WITH CHECK (bucket_id = 'quiz-images');

-- السماح بحذف الصور
CREATE POLICY "Public delete quiz-images"
ON storage.objects FOR DELETE
USING (bucket_id = 'quiz-images');
```

> 💡 **بديل بدون Storage**: إذا لم تنشئ bucket، يمكنك ببساطة إدخال **روابط صور خارجية** في حقول الصور (من imgur, google drive public, إلخ) - الأمر يعمل بشكل طبيعي.

## الجزء 2: تكوين المشروع

### الخطوة 5: تحديث ملف الإعدادات

1. افتح ملف `js/config.js` بمحرر نصوص (مثل VS Code)
2. ابحث عن السطور:

```javascript
SUPABASE_URL: 'https://YOUR_PROJECT_ID.supabase.co',
SUPABASE_ANON_KEY: 'YOUR_SUPABASE_ANON_KEY_HERE',
```

3. استبدلها بالقيم التي نسختها من Supabase:

```javascript
SUPABASE_URL: 'https://abcdefghijk.supabase.co',
SUPABASE_ANON_KEY: 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...',
```

4. **مهم**: غيّر كلمة مرور الإدارة:

```javascript
ADMIN_PASSWORD: 'كلمة_مرور_قوية_خاصة_بك'
```

5. احفظ الملف.

## الجزء 3: التشغيل المحلي للاختبار

### الخطوة 6: تشغيل خادم محلي

افتح **Terminal** أو **CMD** في مجلد المشروع وشغّل:

**Python**:
```bash
python -m http.server 8000
```

**Node.js**:
```bash
npx serve
```

**أو** ببساطة افتح ملف `index.html` بالنقر المزدوج.

### الخطوة 7: اختبار سريع

افتح المتصفح على:
```
http://localhost:8000
```

يجب أن ترى صفحة الدخول.

## الجزء 4: إعداد المحتوى

### الخطوة 8: الدخول للوحة الإدارة

1. افتح: `http://localhost:8000/admin.html`
2. أدخل كلمة المرور التي حددتها في `config.js`
3. ستفتح لوحة الإدارة

### الخطوة 9: إضافة أيام المسابقة

1. من القائمة الجانبية اختر **أيام المسابقة**
2. اضغط **+ إضافة يوم جديد**
3. املأ:
   - رقم اليوم: 1
   - اسم اليوم: "اليوم الأول"
   - التاريخ: تاريخ اليوم
4. اضغط **حفظ**
5. كرر للأيام المطلوبة

### الخطوة 10: رفع قائمة الطلاب

1. اذهب إلى **الطلاب**
2. اضغط **📥 تحميل قالب Excel** للحصول على نموذج فارغ
3. افتح الملف بـ Excel أو Google Sheets
4. املأ بيانات الطلاب:

| الرقم الخاص | اسم الطالب | الشعبة الصفية |
|------------|-----------|---------------|
| 1001 | محمد أحمد | 6/1 |
| 1002 | علي خالد | 6/2 |
| 1003 | يوسف عمر | 6/1 |

5. احفظ الملف
6. ارجع لصفحة الطلاب
7. اضغط على منطقة الرفع واختر الملف
8. تأكد من إضافة الطلاب في الجدول

> 💡 **نصيحة**: يمكنك أيضًا إضافة الطلاب يدوياً بزر **+ إضافة طالب**

### الخطوة 11: رفع الأسئلة

#### تحضير ملف Word:

افتح Microsoft Word واكتب الأسئلة بهذا التنسيق **بالضبط**:

```
السؤال: ما هي عاصمة قطر؟
أ) دبي
ب) الدوحة
ج) الرياض
د) القاهرة
الإجابة: ب

السؤال: كم عدد أركان الإسلام؟
أ) 3
ب) 4
ج) 5
د) 6
الإجابة: ج
```

> 📝 **ملاحظات مهمة لتنسيق الملف**:
> - كل سؤال يبدأ بكلمة "السؤال:"
> - الخيارات تستخدم: أ) ب) ج) د)
> - الإجابة تستخدم: "الإجابة:" متبوعة بحرف (أ/ب/ج/د)
> - اترك سطراً فارغاً بين كل سؤال وآخر
> - احفظ الملف بصيغة `.docx` (وليس `.doc`)

#### رفع الملف:

1. اذهب إلى **الأسئلة** في لوحة الإدارة
2. اضغط على منطقة الرفع واختر ملف Word
3. ستفتح نافذة لمراجعة كل سؤال:
   - اختر **اليوم** الذي ينتمي له السؤال
   - اختر **المادة** (رياضيات، علوم، ...)
   - أضف **رابط الصورة** إذا أردت (اختياري)
   - تأكد من **الإجابة الصحيحة**
4. اضغط **💾 حفظ السؤال**
5. سينتقل تلقائياً للسؤال التالي
6. كرر لكل الأسئلة

> 💡 **نصيحة**: يُفضل أن يكون عدد الأسئلة في اليوم = 18 سؤالاً (3 من كل مادة)

### الخطوة 12: تفعيل اليوم

1. ارجع إلى **أيام المسابقة**
2. اضغط زر **تفعيل** بجانب اليوم المطلوب
3. سيظهر اليوم باللون الأصفر مع علامة 🟢

> ⚠️ يمكن تفعيل يوم واحد فقط في كل مرة

## الجزء 5: النشر على الإنترنت

### الخيار 1: GitHub Pages (الأبسط والمجاني)

1. أنشئ مستودع جديد على https://github.com
2. ارفع جميع ملفات المشروع للمستودع:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO.git
git push -u origin main
```

3. اذهب لـ **Settings** → **Pages**
4. تحت **Source** اختر:
   - Branch: `main`
   - Folder: `/ (root)`
5. اضغط **Save**
6. انتظر دقيقتين وسيظهر رابط الموقع

### الخيار 2: Netlify (سحب وإفلات)

1. اذهب لـ https://netlify.com
2. سجّل الدخول
3. اسحب مجلد المشروع كاملاً إلى منطقة "drag and drop"
4. سيتم النشر فوراً
5. ستحصل على رابط مثل: `https://random-name.netlify.app`

### الخيار 3: Vercel

```bash
npm i -g vercel
cd quiz-project
vercel
```

اتبع التعليمات على الشاشة.

1. شارك رابط الصفحة الرئيسية:

   - يحل الأسئلة
   - يرى نتيجته فوراً
   - يمكنه مشاهدة لوحة الصدارة

## ❓ حل المشاكل الشائعة

### المشكلة: "فشل الاتصال بقاعدة البيانات"

✅ **الحل**:
- تأكد من صحة `SUPABASE_URL` و `SUPABASE_ANON_KEY` في `config.js`
- تأكد أن المشروع في Supabase يعمل (الأخضر في dashboard)

### المشكلة: "لا توجد مسابقة نشطة"

✅ **الحل**:
- اذهب للوحة الإدارة → أيام المسابقة → فعّل يوماً

### المشكلة: "الرقم الخاص غير صحيح"

✅ **الحل**:
- تأكد من رفع الطالب في جدول الطلاب
- تأكد من كتابة الرقم بشكل صحيح (بدون مسافات)

### المشكلة: الأسئلة لا تظهر للطالب

✅ **الحل**:
- تأكد من ربط الأسئلة باليوم النشط
- تأكد من رفع 18 سؤالاً على الأقل (أو عدّل `QUESTIONS_PER_DAY` في config.js)

### المشكلة: ملف Word لا يقرأ الأسئلة

✅ **الحل**:
- تأكد من استخدام صيغة `.docx` (وليس `.doc`)
- تأكد من اتباع التنسيق بدقة (السؤال: / أ) / ب) / ج) / د) / الإجابة:)
- جرب إعادة حفظ الملف من Word
- يمكنك إضافة الأسئلة يدوياً كحل بديل

## 🎓 نصائح للنجاح

1. **اختبر قبل البدء**: حاول حل المسابقة بنفسك للتأكد من سلامة الأسئلة
2. **احتفظ بنسخة احتياطية**: قم بتصدير النتائج من لوحة الإدارة بشكل دوري
3. **استخدم صور واضحة**: اختر صوراً ذات دقة عالية ولكن حجم معقول
4. **راجع الأسئلة**: تحقق من صحة الإجابات قبل بدء المسابقة
5. **جدول الأسئلة**: حاول توزيع 3 أسئلة لكل مادة (= 18 سؤال)
